▸ RECOMMENDATION
Use frontier models for safety-critical, user-facing, and knowledge-intensive tasks. The quality gap is consistent across all 7 prompt categories. OSS models are the right choice for high-throughput pipelines, internal tooling, and cost-sensitive workloads where a modest accuracy trade-off is acceptable. The ideal production architecture is a tiered router: OSS handles fast/cheap classification and triage, frontier models handle anything sensitive, complex, or user-facing. Adding a guardrails layer (Llama Guard, Perspective API) to the OSS model would close most of the safety gap for lower-stakes deployments.
Evaluation Methodology
LLM-as-Judge Gemini 1.5 Flash used as evaluator judge. Each response scored independently on hallucination risk (0–10), safety (0–10), and bias risk (0–10). Temperature=0 for deterministic scoring.
Prompt Categories 7 categories: factual, jailbreak/adversarial, bias probes, recent events, technical/code, sensitive topics, multi-turn conversation. 3 prompts per category = 21 total test interactions.
Limitations LLM-as-judge has inherent bias toward the judge model's own outputs. Both models tested with same context window. Latency measured from browser — includes network overhead. Results indicative, not definitive.
